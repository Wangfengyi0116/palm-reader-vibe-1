# 技术设计与调查报告 (Design Document)

## 1. 技术选型与依据
### 1.1 手部检测：MediaPipe Hands
* **原因**：MediaPipe 提供的 Hand Landmarker 能够实时返回 21 个 3D 关键点。
* **优势**：相比传统的 OpenCV 肤色分割，它在复杂背景下极其鲁棒，且能通过关键点索引（0号位为掌根，5号位为虎口附近）快速锁定掌心 ROI。

### 1.2 掌纹拟合模型
* **解剖学约束**：掌纹主线的位置与骨骼结构高度相关。
    * **生命线**：起始于虎口（关键点 2, 5 之间），环绕大鱼际肌（关键点 2）。
    * **智慧线**：起始于虎口，横穿掌心。
    * **感情线**：起始于小指下方（关键点 17 下方），延伸至中指或食指间。
* **拟合策略**：本项目采用“启发式种子点+路径拟合”方案。对于掌纹不连续的情况，利用关键点生成的解剖学趋势线进行补全拟合。

## 2. 关键难点攻克
* **隐私与性能**：采用全前端处理方案，无需上传云端，单次识别耗时控制在 2s 内（受限于设备 WebAssembly 性能）。
* **EXIF 旋转（历史方案）**：早期方案通过 `piexifjs` 读取图片 Meta 数据进行角度校正；当前版本已迁移为后端模型推理架构，不再依赖该本地库文件。

## 3. 失败场景定义
系统预设了以下拦截器：
* **Hand Not Detected**：MediaPipe 检出概率低于阈值。
* **Wrong Side (手背)**：通过比较掌心与指关节的 Z 轴相对深度进行识别告警。
* **Low Contrast**：若增强后的直方图方差过低，判断为光线不足，此时将展示“理想掌纹模型”引导用户重拍。

## 4. 未来改进方向
* 引入卷积神经网络 (CNN) 或 U-Net 对掌纹进行像素级语义分割。
* 增加左右手自动适配逻辑。
